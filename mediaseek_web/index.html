当前 Web 版无法自动读取你本机浏览器登录态。请先从浏览器导出目标站点 Cookie，再上传到这里解析。

you-get

推荐使用 Firefox 已登录对应站点后，直接上传该浏览器配置目录里的 cookies.sqlite。

常见位置：

  • Windows：%APPDATA%\Mozilla\Firefox\Profiles\<profile>\cookies.sqlite
  • macOS：~/Library/Application Support/Firefox/Profiles/<profile>/cookies.sqlite
  • Linux：~/.mozilla/firefox/<profile>/cookies.sqlite

建议步骤：

  1. 先在 Firefox 中登录目标视频网站。
  2. 打开 about:profiles，找到当前正在使用的配置目录。
  3. 关闭 Firefox，避免数据库还在占用。
  4. 进入对应配置目录，找到 cookies.sqlite。
  5. 把这个文件上传到 MediaSeek。
yt-dlp

推荐从已登录浏览器导出 cookies.txt。如果你手头只有 Firefox 的 cookies.sqlite，也可以直接上传。

cookies.txt 推荐导出方法：

  1. 在浏览器扩展商店安装支持导出 Cookie 的扩展，例如 “Get cookies.txt LOCALLY”。
  2. 打开目标视频网站并确认当前账号已经登录。
  3. 在目标网站页面点击扩展，选择导出当前站点 Cookie。
  4. 保存为 cookies.txt 文件。
  5. 把导出的 cookies.txt 上传到 MediaSeek。

适用浏览器：Chrome、Edge、Brave、Firefox 等常见浏览器。

lux

推荐上传 cookies.txt。如果你使用 Firefox，也可以上传 cookies.sqlite 让后端自动转换。

推荐做法：

  1. 优先按上面的方式导出 cookies.txt。
  2. 如果你已经在 Firefox 登录过目标站点，也可以直接上传 cookies.sqlite。
  3. 遇到需要登录或风控较强的视频网站时，建议使用最新登录态重新导出。
